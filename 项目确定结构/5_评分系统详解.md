# 评分系统详解

## 概述

评分系统用于识别潜在的庄家用户。每个用户的分数范围是 0-100，由 8 个评分规则组成，每个规则最多贡献 10 分。

## 评分流程

```
用户数据 (userInfo + traderStats)
    ↓
ScoringEngine.calculateScores()
    ↓
应用 8 个评分规则
    ├─ 规则 1: 无资金来源 (+10)
    ├─ 规则 2: 同源地址 (+10)
    ├─ 规则 3: 时间聚集 (+10)
    ├─ 规则 4: 高持仓占比 (+10)
    ├─ 规则 5: 高买入金额 (+10)
    ├─ 规则 6: 早期买入 (+10)
    ├─ 规则 7: 多次买入 (+10)
    └─ 规则 8: 手动标记 (+10)
    ↓
返回 scoreMap 和 whaleAddresses
```

## 8 个评分规则

### 规则 1: 无资金来源 (+10)

**配置键**: `enable_no_source`

**判断逻辑**:
```javascript
if (!user.funding_account || user.funding_account === '') {
  score += 10;
  reasons.push('无来源(+10)');
}
```

**说明**:
- 如果用户没有资金来源地址，可能是新创建的钱包
- 庄家常用新钱包来隐藏资金来源

**示例**:
```javascript
// 用户 A: funding_account = null → +10 分
// 用户 B: funding_account = "Binance地址" → 0 分
```

### 规则 2: 同源地址 (+10)

**配置键**: `enable_same_source`

**配置参数**:
- `same_source_n`: 同源数量阈值（默认 5）
- `same_source_exclude`: 排除的来源地址（逗号分隔）

**判断逻辑**:
```javascript
// 统计每个资金来源的用户数量
const sourceCount = {};
for (const user of users) {
  if (user.funding_account) {
    sourceCount[user.funding_account] = (sourceCount[user.funding_account] || 0) + 1;
  }
}

// 如果用户的资金来源有 >= same_source_n 个用户，且不在排除列表中
if (sourceCount[user.funding_account] >= same_source_n && !isExcluded) {
  score += 10;
  reasons.push('同源(+10)');
}
```

**说明**:
- 如果多个用户来自同一个资金来源，可能是庄家的多个钱包
- 可以排除交易所地址（如 Binance、OKX）

**示例**:
```javascript
// 5 个用户都来自 "地址A"
// → 这 5 个用户都 +10 分

// 排除列表: "Binance地址,OKX地址"
// 10 个用户来自 "Binance地址"
// → 这 10 个用户不加分（因为在排除列表中）
```

### 规则 3: 时间聚集 (+10)

**配置键**: `enable_time_cluster`

**配置参数**:
- `time_cluster_n`: 聚集数量阈值（默认 5）
- `time_cluster_j`: 时间窗口（分钟，默认 1）

**判断逻辑**:
```javascript
// 统计在时间窗口内买入的用户数量
const timeWindow = time_cluster_j * 60;  // 转换为秒
const buyTimes = users.map(u => u.first_buy_time).sort();

for (const user of users) {
  const startTime = user.first_buy_time;
  const endTime = startTime + timeWindow;

  // 统计在 [startTime, endTime] 内买入的用户数量
  const count = buyTimes.filter(t => t >= startTime && t <= endTime).length;

  if (count >= time_cluster_n) {
    score += 10;
    reasons.push('时间聚集(+10)');
  }
}
```

**说明**:
- 如果多个用户在短时间内买入，可能是庄家的协同操作
- 时间窗口越小，越能识别精确的时间聚集

**示例**:
```javascript
// 配置: time_cluster_n = 5, time_cluster_j = 1 (1 分钟)
// 用户 A 在 10:00:00 买入
// 用户 B 在 10:00:10 买入
// 用户 C 在 10:00:20 买入
// 用户 D 在 10:00:30 买入
// 用户 E 在 10:00:40 买入
// → 这 5 个用户都在 1 分钟内买入，都 +10 分
```

### 规则 4: 高持仓占比 (+10)

**配置键**: `enable_high_holding`

**配置参数**:
- `high_holding_pct`: 持仓占比阈值（%，默认 10）

**判断逻辑**:
```javascript
if (user.holding_share_pct >= high_holding_pct) {
  score += 10;
  reasons.push('高持仓(+10)');
}
```

**说明**:
- 持仓占比高的用户可能是庄家
- 庄家通常持有大量代币

**示例**:
```javascript
// 配置: high_holding_pct = 10
// 用户 A: holding_share_pct = 15% → +10 分
// 用户 B: holding_share_pct = 5% → 0 分
```

### 规则 5: 高买入金额 (+10)

**配置键**: `enable_high_buy`

**配置参数**:
- `high_buy_amount`: 买入金额阈值（USD，默认 1000）

**判断逻辑**:
```javascript
if (user.total_buy_u >= high_buy_amount) {
  score += 10;
  reasons.push('高买入(+10)');
}
```

**说明**:
- 买入金额高的用户可能是庄家
- 庄家通常投入大量资金

**示例**:
```javascript
// 配置: high_buy_amount = 1000
// 用户 A: total_buy_u = 1500 USD → +10 分
// 用户 B: total_buy_u = 500 USD → 0 分
```

### 规则 6: 早期买入 (+10)

**配置键**: `enable_early_buy`

**配置参数**:
- `early_buy_minutes`: 早期时间窗口（分钟，默认 5）

**判断逻辑**:
```javascript
// 找到最早的买入时间
const earliestBuyTime = Math.min(...users.map(u => u.first_buy_time));

// 如果用户在最早买入时间 + early_buy_minutes 内买入
const timeWindow = early_buy_minutes * 60;
if (user.first_buy_time <= earliestBuyTime + timeWindow) {
  score += 10;
  reasons.push('早期买入(+10)');
}
```

**说明**:
- 早期买入的用户可能是庄家
- 庄家通常在代币刚发行时买入

**示例**:
```javascript
// 配置: early_buy_minutes = 5
// 最早买入时间: 10:00:00
// 用户 A: first_buy_time = 10:02:00 → +10 分（在 5 分钟内）
// 用户 B: first_buy_time = 10:10:00 → 0 分（超过 5 分钟）
```

### 规则 7: 多次买入 (+10)

**配置键**: `enable_multiple_buy`

**配置参数**:
- `multiple_buy_count`: 买入次数阈值（默认 3）

**判断逻辑**:
```javascript
// 统计用户的买入次数（从交易历史中）
const buyCount = traderHistory[user.owner].filter(tx => tx.action === 'buy').length;

if (buyCount >= multiple_buy_count) {
  score += 10;
  reasons.push('多次买入(+10)');
}
```

**说明**:
- 多次买入的用户可能是庄家
- 庄家可能分批买入以隐藏意图

**示例**:
```javascript
// 配置: multiple_buy_count = 3
// 用户 A: 买入 4 次 → +10 分
// 用户 B: 买入 2 次 → 0 分
```

### 规则 8: 手动标记 (+10)

**配置键**: 无（始终启用）

**配置参数**:
- `manual_score`: 手动标记分数（默认 10）

**判断逻辑**:
```javascript
if (manualScores[user.owner] === '庄家') {
  score += manual_score;
  reasons.push('手动标记(+10)');
}
```

**说明**:
- 用户可以手动标记某个地址为庄家
- 手动标记会增加固定分数

**示例**:
```javascript
// 用户手动标记 "地址A" 为庄家
// → "地址A" +10 分
```

## 状态判断

### 散户 vs 庄家

**判断逻辑**:
```javascript
if (score >= statusThreshold) {
  status = '庄家';
} else {
  status = '散户';
}
```

**默认阈值**: `statusThreshold = 50`

**示例**:
```javascript
// statusThreshold = 50
// 用户 A: score = 60 → status = "庄家"
// 用户 B: score = 40 → status = "散户"
// 用户 C: score = 50 → status = "庄家"
```

## 评分配置建议

### 保守配置（减少误判）

```javascript
{
  enable_no_source: true,
  enable_same_source: true,
  enable_time_cluster: true,
  enable_high_holding: false,
  enable_high_buy: false,
  enable_early_buy: false,
  enable_multiple_buy: false,
  same_source_n: 10,              // 提高阈值
  time_cluster_n: 10,             // 提高阈值
  time_cluster_j: 1,
  statusThreshold: 60             // 提高阈值
}
```

### 激进配置（提高识别率）

```javascript
{
  enable_no_source: true,
  enable_same_source: true,
  enable_time_cluster: true,
  enable_high_holding: true,
  enable_high_buy: true,
  enable_early_buy: true,
  enable_multiple_buy: true,
  same_source_n: 3,               // 降低阈值
  time_cluster_n: 3,              // 降低阈值
  time_cluster_j: 2,
  high_holding_pct: 5,            // 降低阈值
  high_buy_amount: 500,           // 降低阈值
  early_buy_minutes: 10,          // 扩大窗口
  multiple_buy_count: 2,          // 降低阈值
  statusThreshold: 40             // 降低阈值
}
```

## 注意事项

1. **规则独立性**: 每个规则独立计算，不会相互影响
2. **分数累加**: 用户的总分是所有触发规则的分数之和
3. **最大分数**: 理论最大分数是 80 分（8 个规则 × 10 分）
4. **阈值调整**: statusThreshold 可以根据实际情况调整
5. **配置持久化**: 所有配置存储在 Chrome Storage 中
6. **实时更新**: 配置变化时会立即重新评分
