# 配置数据结构

## 概述

插件的所有配置存储在 Chrome Storage Local 中，包括评分配置、阈值设置、手动标记等。配置变化时会实时同步到 HeliusMonitor。

## 存储位置

所有配置存储在 `chrome.storage.local` 中。

## 配置项列表

### 1. 评分配置 (boss_config)

**存储键**: `boss_config`

**数据结构**:
```javascript
{
  // 规则 1: 无资金来源
  enable_no_source: true,              // 是否启用
  no_source_score: 10,                 // 分数 (默认 10)

  // 规则 2: 同源地址
  enable_same_source: false,           // 是否启用
  same_source_n: 5,                    // 同源数量阈值
  same_source_exclude: "",             // 排除的来源地址 (逗号分隔)
  same_source_score: 10,               // 分数 (默认 10)

  // 规则 3: 时间聚集
  enable_time_cluster: false,          // 是否启用
  time_cluster_n: 5,                   // 聚集数量阈值
  time_cluster_j: 1,                   // 时间窗口 (分钟)
  time_cluster_score: 10,              // 分数 (默认 10)

  // 规则 4: 高持仓占比
  enable_high_holding: false,          // 是否启用
  high_holding_pct: 10,                // 持仓占比阈值 (%)
  high_holding_score: 10,              // 分数 (默认 10)

  // 规则 5: 高买入金额
  enable_high_buy: false,              // 是否启用
  high_buy_amount: 1000,               // 买入金额阈值 (USD)
  high_buy_score: 10,                  // 分数 (默认 10)

  // 规则 6: 早期买入
  enable_early_buy: false,             // 是否启用
  early_buy_minutes: 5,                // 早期时间窗口 (分钟)
  early_buy_score: 10,                 // 分数 (默认 10)

  // 规则 7: 多次买入
  enable_multiple_buy: false,          // 是否启用
  multiple_buy_count: 3,               // 买入次数阈值
  multiple_buy_score: 10,              // 分数 (默认 10)

  // 规则 8: 手动标记
  manual_score: 10                     // 手动标记分数 (默认 10)
}
```

**默认值**:

```javascript
  high_buy_score: 10,
  early_buy_score: 10,
  multiple_buy_score: 10,
  manual_score: 10

```

### 2. Score< 阈值 (score_threshold)

**存储键**: `score_threshold`

**数据类型**: `number`

**默认值**: `100`

**说明**:
- 只有 score < score_threshold 的用户才会被显示
- 范围: 0-100
- 用户可以在设置页面修改

**示例**:
```javascript
// 显示所有用户
score_threshold = 100

// 只显示分数 < 50 的用户
score_threshold = 50

// 只显示分数 < 30 的用户
score_threshold = 30
```

### 3. 状态阈值 (status_threshold)

**存储键**: `status_threshold`

**数据类型**: `number`

**默认值**: `50`

**说明**:
- score >= status_threshold 的用户状态为"庄家"
- score < status_threshold 的用户状态为"散户"
- 范围: 0-100

**示例**:
```javascript
// 用户 A: score = 60 → status = "庄家"
// 用户 B: score = 40 → status = "散户"
status_threshold = 50
```

### 4. 手动标记 (manual_scores_{mint})

**存储键**: `manual_scores_{mint}` (每个 mint 独立存储)

**数据结构**:
```javascript
{
  "8F5pDmMVfKVQUU2TtjehFDkKqMKKQpxr2TTaYxwK6bFG": "庄家",
  "DbzUoXKGvVXfpe8yx4E4MHJbPXoUh4Yx8Yx4E4MHJbPX": "散户"
}
```

**说明**:
- key: 用户钱包地址
- value: 手动标记的状态 ("庄家" 或 "散户")
- 手动标记为"庄家"会增加 10 分

### 5. Helius API 配置

**存储键**: `helius_api_key`

**数据类型**: `string`

**说明**: Helius API 密钥，用于实时监听链上交易

### 6. 其他配置

| 存储键 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `holder_limit` | number | 100 | 每页获取的 holder 数量 |
| `holder_max` | number | 1000 | 最多获取的 holder 数量 |
| `env_keys` | array | [] | GMGN API 密钥列表 |

## 配置加载流程

### 初始化加载

```javascript
// HeliusIntegration.init()
async init() {
  const res = await chrome.storage.local.get([
    'boss_config',
    'score_threshold',
    'status_threshold',
    `manual_scores_${this.currentMint}`
  ]);

  this.bossConfig = res.boss_config || this.getDefaultConfig();
  this.scoreThreshold = res.score_threshold !== undefined ? res.score_threshold : 100;
  this.statusThreshold = res.status_threshold !== undefined ? res.status_threshold : 50;
  this.manualScores = res[`manual_scores_${this.currentMint}`] || {};
}
```

### 配置变化监听

```javascript
// HeliusIntegration 监听配置变化
chrome.storage.onChanged.addListener((changes, area) => {
  if (area === 'local') {
    if (changes.score_threshold) {
      this.scoreThreshold = changes.score_threshold.newValue;
      this.monitor.setScoreThreshold(this.scoreThreshold);
      this.sendDataToSidepanel();  // 重新发送过滤后的数据
    }

    if (changes.status_threshold) {
      this.statusThreshold = changes.status_threshold.newValue;
      this.monitor.setStatusThreshold(this.statusThreshold);
      // 需要重新评分
    }

    if (changes.boss_config) {
      this.bossConfig = changes.boss_config.newValue;
      this.monitor.setBossConfig(this.bossConfig);
      // 需要重新评分
    }
  }
});
```

## 配置修改流程

### 用户修改 Score< 阈值

1. **用户在设置页面修改** → 输入新值
2. **保存到 Chrome Storage** → `chrome.storage.local.set({ score_threshold: newValue })`
3. **触发 onChanged 事件** → HeliusIntegration 监听到变化
4. **更新 HeliusMonitor** → `monitor.setScoreThreshold(newValue)`
5. **重新过滤用户** → `filterUsersByScore()`
6. **发送更新数据** → `sendDataToSidepanel()`
7. **UI 实时更新** → 用户列表刷新

### 用户修改评分配置

1. **用户在设置页面修改** → 启用/禁用规则或修改参数
2. **保存到 Chrome Storage** → `chrome.storage.local.set({ boss_config: newConfig })`
3. **触发 onChanged 事件** → HeliusIntegration 监听到变化
4. **更新 HeliusMonitor** → `monitor.setBossConfig(newConfig)`
5. **重新评分** → 需要重新调用 `updateHolderData()`
6. **发送更新数据** → `sendDataToSidepanel()`
7. **UI 实时更新** → 用户列表刷新

## 配置持久化

所有配置都持久化存储在 Chrome Storage Local 中，具有以下特点：

1. **跨会话持久化**: 关闭浏览器后配置不会丢失
2. **跨页面同步**: 所有页面共享同一份配置
3. **实时同步**: 配置变化时所有监听器都会收到通知

## 配置优先级

1. **Chrome Storage** (最高优先级)
2. **代码中的默认值** (备用)

## 注意事项

1. **默认值一致性**: 确保所有地方的默认值一致
   - score_threshold: 100
   - status_threshold: 50
   - boss_config: 见上述默认值

2. **配置验证**: 加载配置时需要验证类型和范围
   ```javascript
   this.scoreThreshold = res.score_threshold !== undefined ? res.score_threshold : 100;
   ```

3. **手动标记隔离**: 每个 mint 的手动标记独立存储，避免混淆
   ```javascript
   `manual_scores_${this.currentMint}`
   ```

4. **配置变化处理**: 某些配置变化需要重新评分，某些只需要重新过滤
   - score_threshold 变化 → 只需重新过滤
   - boss_config 变化 → 需要重新评分
   - status_threshold 变化 → 需要重新评分
