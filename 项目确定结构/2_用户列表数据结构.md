# 用户列表数据结构

## 概述

用户列表显示在插件的 Sidepanel 中，展示所有符合 Score< 阈值的用户信息。数据由 HeliusIntegration 发送，UI 接收并渲染。

## 数据流

```
HeliusMonitor.metricsEngine.userInfo (完整用户信息)
    ↓
HeliusMonitor.metricsEngine.filteredUsers (过滤后的地址集合)
    ↓
HeliusIntegration.sendDataToSidepanel() (筛选并格式化)
    ↓
chrome.runtime.sendMessage({ type: 'UI_RENDER_DATA', data: holdersData })
    ↓
Sidepanel App.jsx 接收并显示
```

## 字段说明

### 核心字段

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| `owner` | string | 用户钱包地址（完整地址） | "8F5pDmMVfKVQUU2TtjehFDkKqMKKQpxr2TTaYxwK6bFG" |
| `score` | number | 用户评分 (0-100) | 20 |
| `status` | string | 用户状态 | "散户" 或 "庄家" |
| `score_reasons` | array | 评分原因列表 | ["无来源(+10)", "同源(+10)"] |

### 持仓信息

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| `ui_amount` | number | 持仓数量（浮点数） | 1000000 |
| `holding_share_pct` | number | 持仓占比 (%) | 5.2 |
| `total_buy_u` | number | 总买入金额 (USD) | 100 |

### 资金来源

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| `funding_account` | string \| null | 资金来源地址 | "Binance地址" 或 null |
| `first_buy_time` | number | 首次买入时间戳 | 1234567890 |

### 元数据

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| `data_source` | string | 数据来源标识 | "GMGN Holder API" |

## UI 显示格式

### 地址显示

完整地址会在 UI 中缩写显示：
```javascript
// 完整地址: "8F5pDmMVfKVQUU2TtjehFDkKqMKKQpxr2TTaYxwK6bFG"
// 显示为: "8F5pDmMV..."
const shortAddress = owner.substring(0, 8) + '...';
```

### 分数显示

- 分数范围: 0-100
- 颜色标识:
  - 0-29: 绿色 (散户)
  - 30-49: 黄色 (可疑)
  - 50-100: 红色 (庄家)

### 状态显示

- **散户**: score < statusThreshold (默认 50)
- **庄家**: score >= statusThreshold (默认 50)

## 过滤逻辑

### Score< 过滤

只有满足 `score < scoreThreshold` 的用户才会被发送给 UI。

**示例**:
- scoreThreshold = 30
- 用户 A: score = 20 → ✅ 显示
- 用户 B: score = 40 → ❌ 不显示

### 代码实现

```javascript
// HeliusIntegration.sendDataToSidepanel()
const userInfo = this.monitor.metricsEngine.userInfo;
const filteredUsers = this.monitor.metricsEngine.filteredUsers;

const holdersData = Object.entries(userInfo)
  .filter(([address]) => filteredUsers.has(address))  // 只保留过滤后的用户
  .map(([, info]) => ({
    ...info,
    status: info.status || '散户',
    score: info.score || 0,
    score_reasons: info.score_reasons || []
  }));
```

## UI 组件接收数据

### App.jsx 消息处理

```javascript
// src/sidepanel/App.jsx
useEffect(() => {
  const handleMessage = (message) => {
    if (message.type === 'UI_RENDER_DATA') {
      const holdersData = message.data;

      // 更新用户列表状态
      setHolders(holdersData);

      console.log('[Sidepanel] 接收到用户数据', {
        count: holdersData.length,
        sample: holdersData[0]
      });
    }
  };

  chrome.runtime.onMessage.addListener(handleMessage);

  return () => {
    chrome.runtime.onMessage.removeListener(handleMessage);
  };
}, []);
```

## 注意事项

1. **地址唯一性**: owner 字段是用户的唯一标识
2. **过滤实时性**: Score< 阈值变化时，UI 会立即收到更新后的用户列表
3. **默认值处理**: status、score、score_reasons 都有默认值，避免 undefined
4. **数据完整性**: 所有字段都来自 MetricsEngine.userInfo，确保数据一致性

## 混合数据源字段（v2.0 新增）

### 数据模式字段

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| `data_mode` | string | 数据模式 | "holder_based" 或 "trade_calculated" |
| `has_holder_snapshot` | boolean | 是否有 Holder 快照 | true/false |
| `last_holder_update` | number | 最后 Holder 更新时间 | 1708435200000 |
| `last_trade_update` | number | 最后 Trade 更新时间 | 1708438800000 |
| `calculated_balance` | number | 从 Trade 计算的持仓 | 1000000 |

### 数据模式说明

#### holder_based 模式

- **特点**: 有完整的 Holder API 数据
- **持仓信息**: 准确（来自 Holder API 快照）
- **Trade 数据**: 增量更新到快照上
- **适用场景**: 用户在 Holder API 返回的前 100 名内

**示例**:
```json
{
  "owner": "8F5pDmMVfKVQUU2TtjehFDkKqMKKQpxr2TTaYxwK6bFG",
  "ui_amount": 1000000,
  "data_source": "GMGN Holder API",
  "data_mode": "holder_based",
  "has_holder_snapshot": true,
  "last_holder_update": 1708435200000,
  "last_trade_update": 1708438800000
}
```

#### trade_calculated 模式

- **特点**: 只有 Trade 数据，没有 Holder 快照
- **持仓信息**: 从交易记录计算（可能不准确）
- **Trade 数据**: 每次新交易都从头计算
- **适用场景**: 用户不在 Holder API 返回的前 100 名内
- **注意**: 如果有链下转账，持仓计算可能不准确

**示例**:
```json
{
  "owner": "NewUser123456789ABCDEFGHIJKLMNOPQRSTUVWXYZ",
  "ui_amount": 500000,
  "calculated_balance": 500000,
  "data_source": "Trade Calculated",
  "data_mode": "trade_calculated",
  "has_holder_snapshot": false,
  "last_trade_update": 1708438800000
}
```

### 模式切换

当用户从 trade_calculated 模式切换到 holder_based 模式时（例如，用户持仓增加进入前 100 名）：

1. `data_mode` 从 "trade_calculated" 变为 "holder_based"
2. `has_holder_snapshot` 从 false 变为 true
3. `ui_amount` 更新为 Holder API 的准确值
4. 添加 `last_holder_update` 时间戳
5. 之后的 Trade 数据使用增量更新

**切换示例**:
```json
// 切换前
{
  "owner": "User123...",
  "ui_amount": 500000,
  "data_mode": "trade_calculated",
  "has_holder_snapshot": false
}

// 切换后
{
  "owner": "User123...",
  "ui_amount": 1200000,  // 更准确的值
  "data_mode": "holder_based",
  "has_holder_snapshot": true,
  "last_holder_update": 1708435200000
}
```

### 数据来源标识更新

`data_source` 字段现在有两种可能的值：

| 值 | 说明 | 对应模式 |
|----|------|----------|
| "GMGN Holder API" | 数据来自 Holder API | holder_based |
| "Trade Calculated" | 数据从 Trade 计算 | trade_calculated |

### UI 显示建议

在用户详情面板中，可以根据 `data_mode` 显示不同的提示：

- **holder_based**: "数据来源：Holder API（准确）"
- **trade_calculated**: "数据来源：Trade 计算（可能不准确）"

### 向后兼容性

- 所有新增字段都是可选的
- 旧版本的数据不包含这些字段，UI 应该能正常显示
- 新字段不影响现有的评分、过滤和显示逻辑
