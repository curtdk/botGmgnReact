## 核心概念

### 唯一数据源

**HeliusMonitor** 是唯一的数据处理和存储中心。所有数据从 GMGN API 获取后，统一由 HeliusMonitor 处理、评分、过滤，然后分发给 UI。

**详见**: [数据源架构](./1_数据源架构.md)

### 提前过滤

在数据源中根据 Score< 阈值提前过滤，然后发送给 UI。UI 不需要再次过滤。

**详见**: [数据源架构](./1_数据源架构.md) + [用户列表数据结构](./2_用户列表数据结构.md)

### 配置持久化

所有配置从 Chrome Storage 加载，确保一致性。配置变化时会实时同步到 HeliusMonitor。

**详见**: [配置数据结构](./3_配置数据结构.md)

### 评分系统

使用 8 个评分规则计算用户分数，分数范围 0-100。score >= statusThreshold 的用户状态为"庄家"。

**详见**: [评分系统详解](./5_评分系统详解.md)
