## 根本原因

### BossLogic.js 的评分逻辑

从 [BossLogic.js:14-28](../src/content/BossLogic.js#L14-L28) 可以看到:

```javascript
static calculateUserScore(user, stats, config) {
    let score = 0;
    let isBoss = false;
    const reasons = [];

    if (!config) return { score, isBoss, reasons };

    // 1. 无资金来源 (Funding Account 为空)
    if (!user.funding_account) {
        score += (config.weight_no_source || 0);
        reasons.push('无来源');
        if (config.enable_no_source) {
            isBoss = true;
        }
    }
    // ... 其他规则
}
```

**关键点**:
1. 如果 `config` 为空或未定义,返回分数 0
2. 每个评分规则需要对应的配置项(如 `weight_no_source`, `enable_no_source`)
3. 如果配置项不存在,`config.weight_no_source || 0` 会返回 0

## 可能的问题原因

### 原因 1: Chrome Storage 中的配置被覆盖

**症状**: boss_config 只有 1 个键

**可能原因**:
- 某个地方调用了 `chrome.storage.local.set({ boss_config: {...} })` 但只设置了部分配置
- 配置页面保存配置时只保存了部分字段

**解决方案**:
- 检查所有调用 `chrome.storage.local.set` 的地方
- 确保保存配置时使用完整的配置对象
- 使用合并配置的方案(修复 3)

### 原因 2: 配置从未初始化

**症状**: Chrome Storage 中没有 boss_config

**可能原因**:
- 用户首次使用插件,配置还未初始化
- 配置被清除或重置

**解决方案**:
- 在 HeliusIntegration.init() 中检查配置是否存在
- 如果不存在,保存默认配置到 Chrome Storage
- 使用 `|| this.getDefaultConfig()` 确保有默认值

### 原因 3: 配置格式错误

**症状**: boss_config 存在但格式不对

**可能原因**:
- 配置被保存为字符串而不是对象
- 配置结构不正确

**解决方案**:
- 添加配置验证逻辑
- 如果配置格式错误,使用默认配置
- 添加日志显示配置类型和内容

## 推荐修复方案

### 方案 1: 合并配置 + 初始化检查(推荐)

**文件**: `/www/wwwroot/py/pumpfunbot/gmgn-extension-react/src/content/HeliusIntegration.js:51-67`

```javascript
async init() {
  // 从 storage 加载开关状态和配置
  chrome.storage.local.get([
    'helius_monitor_enabled',
    'boss_config',
    'score_threshold',
    'status_threshold'
  ], (res) => {
    this.enabled = res.helius_monitor_enabled || false;

    // 合并配置:使用默认配置作为基础,然后覆盖用户配置
    const defaultConfig = this.getDefaultConfig();
    this.bossConfig = {
      ...defaultConfig,
      ...(res.boss_config || {})
    };

    // 如果 Chrome Storage 中没有配置,保存默认配置
    if (!res.boss_config) {
      console.log('[Helius集成] 首次初始化,保存默认配置');
      chrome.storage.local.set({ boss_config: defaultConfig });
    }

    this.scoreThreshold = res.score_threshold || 100;
    this.statusThreshold = res.status_threshold || 50;

    console.log('[Helius集成] 开关状态:', this.enabled ? '启用' : '禁用');
    console.log('[Helius集成] Score< 阈值:', this.scoreThreshold);
    console.log('[Helius集成] 状态判断阈值:', this.statusThreshold);
    console.log('[Helius集成] Boss配置:', this.bossConfig);
    console.log('[Helius集成] Boss配置键数量:', Object.keys(this.bossConfig).length);
  });

  // ... 其他初始化代码
}
```

**优点**:
- ✅ 确保配置完整性
- ✅ 保留用户自定义配置
- ✅ 自动初始化默认配置
- ✅ 添加详细日志

## 总结

问题的根源是 **boss_config 配置不完整**,导致所有用户的分数都被计算为 0。当所有用户分数都是 0 时,设置 Score< 10 会显示所有用户(因为 0 < 10),这就是用户看到"分数为 20 的用户也显示"的原因(实际上那些用户的分数也是 0,只是 UI 可能有缓存或显示问题)。

通过添加配置加载日志和使用配置合并方案,可以确保评分系统正常工作。
