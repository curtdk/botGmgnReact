# Helius Monitor 日志格式规范

## 概述

本文档定义了 Helius Monitor 的统一日志格式，确保日志清晰、结构化、易于追踪。

## 设计原则

1. **层次化**：使用阶段、交易、用户三级结构
2. **编号化**：所有关键操作都有序号
3. **状态追踪**：实时显示用户状态变化
4. **数据溯源**：明确标注数据来源
5. **完整性**：显示每个用户的完整交易历史

## 实施要点

### 1. 使用 LogFormatter 工具类

```javascript
import LogFormatter from './LogFormatter.js';

// 阶段开始
LogFormatter.phaseStart('首次计算开始', {
  '总交易数': '13 笔',
  '时间范围': '2026/2/19 14:32:27 - 2026/2/19 14:32:35',
  '数据来源': 'Helius API (13), GMGN (0)'
});

// 交易处理
LogFormatter.transactionStart({
  signature: '3AaYWkzZ...',
  timestamp: '2026/2/19 14:32:27',
  total: 13,
  current: 1
});

// 用户操作
LogFormatter.userAction({
  address: 'GGjze3ST...',
  isNew: true,
  action: '买入',
  solChange: -1.259864,
  tokenChange: 42411067.15,
  source: 'Helius API (initial)'
});

// 用户状态更新
LogFormatter.userStateUpdate({
  address: 'GGjze3ST...',
  totalBuy: 1.259864,
  totalSell: 0,
  tokenBalance: 42411067.15,
  netCost: 1.259864,
  status: 'active'
});

// 阶段结束
LogFormatter.phaseEnd('首次计算完成', {
  '处理交易数': '13 笔',
  '新增用户': '9 个',
  '耗时': '1.5 秒'
});
```

### 2. 在 MetricsEngine 中添加交易历史记录

```javascript
class MetricsEngine {
  constructor() {
    this.traderStats = {};
    this.traderHistory = {}; // 新增：记录每个用户的交易历史
  }

  updateTraderState(address, solChange, tokenChange, signature, timestamp, source) {
    // 更新统计
    if (!this.traderStats[address]) {
      this.traderStats[address] = {
        netSolSpent: 0,
        netTokenReceived: 0,
        totalBuySol: 0,
        totalSellSol: 0
      };
      this.traderHistory[address] = []; // 初始化历史记录
    }

    // 记录交易历史
    this.traderHistory[address].push({
      signature,
      timestamp,
      solChange,
      tokenChange,
      source,
      action: solChange < 0 ? '买入' : '卖出'
    });

    // ... 更新统计逻辑
  }

  // 打印用户历史
  printUserHistory(address) {
    const history = this.traderHistory[address] || [];
    LogFormatter.userHistory({
      address,
      transactions: history
    });
  }
}
```

### 3. 在 HeliusMonitor 中使用阶段管理

```javascript
async performInitialCalculation() {
  LogFormatter.phaseStart('首次计算开始', {
    '总交易数': `${readySignatures.length} 笔`,
    '处理顺序': '按时间倒序（从最早到最新）'
  });

  let processedCount = 0;
  for (const item of readySignatures) {
    processedCount++;

    LogFormatter.transactionStart({
      signature: item.sig,
      timestamp: item.timestamp,
      total: readySignatures.length,
      current: processedCount
    });

    this.metricsEngine.processTransaction(item.txData, this.mint);
    this.signatureManager.markProcessed(item.sig);
  }

  LogFormatter.phaseEnd('首次计算完成', {
    '处理交易数': `${processedCount} 笔`,
    '耗时': `${duration} 秒`
  });

  LogFormatter.metricsSummary(this.metricsEngine.getMetrics());
}
```

## 优势

1. **清晰的层次结构**：通过阶段、交易、用户三级结构，清晰展示处理流程
2. **完整的数据追踪**：每个用户的完整交易历史都被记录和显示
3. **实时状态更新**：每次交易后立即显示用户状态变化
4. **数据来源透明**：明确标注每笔数据的来源（Helius API、WebSocket、GMGN）
5. **易于调试**：结构化的日志便于快速定位问题
