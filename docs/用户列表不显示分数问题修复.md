# 用户列表不显示分数问题修复

## 问题描述

用户报告：
- Helius 实时指标和用户列表都显示了
- 但用户列表中的 Score 分数一直不显示
- 日志显示评分流程完全正常

### 根本原因

**问题在于数据传递方式**：

1. **HeliusIntegration 使用 window 事件**：
   ```javascript
   window.dispatchEvent(new CustomEvent('HELIUS_DATA_UPDATE', {
     detail: { holders: holdersData }
   }));
   ```

2. **Sidepanel App.jsx 无法接收 window 事件**：
   - Content Script 和 Sidepanel 是不同的上下文
   - Sidepanel 无法监听 Content Script 的 window 事件
   - 需要使用 Chrome 消息 API (`chrome.runtime.sendMessage`)

3. **App.jsx 监听的是 `UI_RENDER_DATA` 消息**：
   ```javascript
   const handleMessage = (request, sender, sendResponse) => {
     if (request.type === 'UI_RENDER_DATA') {
       setItems([...request.data]);
     }
   }
   ```

**结果**：评分数据无法传递到 Sidepanel，所以 UI 不显示分数。

## 修复方案

### 修改 HeliusIntegration.distributeDataToContentManager()

**文件**: `/www/wwwroot/py/pumpfunbot/gmgn-extension-react/src/content/HeliusIntegration.js`

**修改位置**: Line 725-733

**修改内容**:

```javascript
// 分发事件给 contentManager（保留用于兼容）
window.dispatchEvent(new CustomEvent('HELIUS_DATA_UPDATE', {
  detail: {
    holders: holdersData,
    statusMap: this.statusMap || {}
  }
}));

// [新增] 发送 Chrome 消息给 sidepanel
chrome.runtime.sendMessage({
  type: 'UI_RENDER_DATA',
  data: holdersData,
  url: null
}).catch(err => {
  // 忽略错误（sidepanel 可能未打开）
  console.log('[HeliusIntegration] 发送消息到 sidepanel 失败（可能未打开）:', err.message);
});
```

**说明**:
1. 保留原有的 window 事件（用于兼容其他可能的监听器）
2. 新增 Chrome 消息发送给 sidepanel
3. 使用 `UI_RENDER_DATA` 类型（App.jsx 已经监听此类型）
4. 数据包含完整的 holdersData（包括 score, status, score_reasons）
5. 使用 `.catch()` 捕获错误（sidepanel 未打开时会失败）

## 技术细节

### Chrome 扩展上下文隔离

Chrome 扩展有多个独立的上下文：

1. **Content Script**：注入到网页中，可以访问 DOM
2. **Sidepanel**：独立的 UI 页面，无法访问网页 DOM
3. **Background Script**：后台服务

**通信方式**：
- ❌ 不能使用 window 事件跨上下文通信
- ✅ 必须使用 Chrome 消息 API：
  - `chrome.runtime.sendMessage()` - 发送消息
  - `chrome.runtime.onMessage.addListener()` - 接收消息

### 为什么保留 window 事件？

虽然 Sidepanel 无法接收 window 事件，但保留它是为了：
1. **向后兼容**：可能有其他代码监听此事件
2. **调试方便**：可以在 Content Script 上下文中监听和调试
3. **未来扩展**：如果需要在同一上下文中通信

## 常见问题

### Q1: 为什么之前没有发现这个问题？

**A**: 因为之前可能使用的是其他数据源（如 index.jsx 的 hook 数据），而不是 HeliusIntegration 的数据。HeliusIntegration 是新集成的数据源，使用了不同的通信方式。

### Q2: 如果 Sidepanel 未打开会怎样？

**A**: `chrome.runtime.sendMessage()` 会失败，但被 `.catch()` 捕获，不会影响其他功能。日志会显示"发送消息到 sidepanel 失败（可能未打开）"。

### Q3: 数据会重复发送吗？

**A**: 不会。虽然同时发送了 window 事件和 Chrome 消息，但：
- window 事件：只有 Content Script 上下文中的监听器会接收
- Chrome 消息：只有 Sidepanel 会接收
- 它们是不同的接收者，不会重复

### Q4: 为什么使用 `UI_RENDER_DATA` 而不是 `HELIUS_DATA_UPDATE`？

**A**: 因为 App.jsx 已经有 `UI_RENDER_DATA` 的处理逻辑（line 556），直接复用可以避免修改 App.jsx。如果使用新的消息类型，需要在 App.jsx 中添加新的处理逻辑。

通过添加 Chrome 消息发送，解决了 Content Script 和 Sidepanel 之间的通信问题：

1. ✅ 评分数据现在可以正确传递到 Sidepanel
2. ✅ 用户列表会显示 Score 分数
