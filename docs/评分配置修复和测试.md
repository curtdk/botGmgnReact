## 修复内容

### 1. 修复配置加载逻辑

**文件**: [src/content/HeliusIntegration.js:51-99](../src/content/HeliusIntegration.js#L51-L99)

**问题**:
- boss_config 只有 1 个键,应该有 16 个
- 配置加载不完整导致所有用户分数都是 0

**修复方案**:
```javascript
// 合并配置:使用默认配置作为基础,然后覆盖用户配置
const defaultConfig = this.getDefaultConfig();
this.bossConfig = {
  ...defaultConfig,
  ...(res.boss_config || {})
};

// 深度合并对象类型的规则配置
['rule_gas', 'rule_amount_sim', 'rule_large_holding', 'rule_sol_balance', 'rule_source_time'].forEach(key => {
  if (res.boss_config && res.boss_config[key]) {
    this.bossConfig[key] = { ...defaultConfig[key], ...res.boss_config[key] };
  }
});

// 如果 Chrome Storage 中没有配置,保存默认配置
if (!res.boss_config) {
  console.log('[Helius集成] 首次初始化,保存默认配置');
  chrome.storage.local.set({ boss_config: defaultConfig });
}
```

**效果**:
- ✅ 确保配置完整性(16个键)
- ✅ 保留用户自定义配置
- ✅ 自动初始化默认配置
- ✅ 深度合并对象类型的规则

getEnabledRules() {

  if (this.bossConfig.rule_source_time?.enabled) enabled.push('同源时间');

## 8个评分规则

### 默认配置

```javascript
{
  // 1. 无资金来源 (默认启用)
  enable_no_source: true,
  weight_no_source: 10,

  // 2. 同源账户 (默认禁用)
  enable_same_source: false,
  same_source_n: 5,
  same_source_exclude: '',
  weight_same_source: 10,

  // 3. 时间聚类 (默认禁用)
  enable_time_cluster: false,
  time_cluster_n: 5,
  time_cluster_j: 1,
  weight_time_cluster: 10,

  // 4. Gas 异常 (默认禁用)
  rule_gas: { enabled: false, threshold: 0.01, weight: 10 },

  // 5. 金额相似 (默认禁用)
  rule_amount_sim: { enabled: false, count: 5, range: 100, weight: 10 },

  // 6. 大额持仓 (默认禁用)
  rule_large_holding: { enabled: false, top_pct: 10, min_usd: 1000, logic: 'OR', weight: 10 },

  // 7. SOL 余额 (默认禁用)
  rule_sol_balance: { enabled: false, count: 3, range: 0.1, weight: 10 },

  // 8. 同源时间 (默认禁用)
  rule_source_time: { enabled: false, diff_sec: 10, count: 2, weight: 10 }
}
```

### 规则说明

1. **无资金来源**: Funding Address 为空的账户 (+10分)
2. **同源账户**: 同一个 Funding Address 的账户数 >= N (+10分)
3. **时间聚类**: 在 J 秒内创建 >= N 个账户 (+10分)
4. **Gas 异常**: 单笔交易 Gas 费 < 阈值 (+10分)
5. **金额相似**: 相似金额(范围±X)的交易数 > N (+10分)
6. **大额持仓**: 前 X% 或持仓 > Y USD (+10分)
7. **SOL 余额**: 余额相似(范围±X)的账户数 > N (+10分)
8. **同源时间**: 来源时间相差 <= X 秒且数量 >= N (+10分)

## 常见问题

### Q1: 为什么所有用户分数都是 0?

**原因**: 配置不完整或所有规则都未启用

**解决方案**:
1. 检查日志中的 "Boss配置键数量",应该是 16
2. 检查 "启用的规则",至少应该有 "无资金来源"
3. 如果配置键数量 < 16,清除配置并刷新页面

### Q2: 为什么启用规则后分数没有变化?

**原因**: 配置保存后没有重新计算分数

**解决方案**:
1. 保存配置后刷新页面
2. 重新启动持有人监听
3. 查看日志确认配置已更新

### Q3: 为什么 Score< 过滤不生效?

**原因**: 所有用户分数都 < 阈值

**解决方案**:
1. 查看所有用户的实际分数
2. 设置合适的阈值
3. 启用更多规则以增加分数差异

### Q4: 如何重置配置?

**方法 1**: 在控制台执行
```javascript
chrome.storage.local.remove('boss_config', () => {
  console.log('已清除配置');
  location.reload();
});
```

**方法 2**: 在庄家设置页面
1. 取消所有规则
2. 恢复默认参数
3. 保存配置
