# 数据源分数修复完成

## 修复目标

确保用户列表只能从数据源（HeliusMonitor）获取分数，不能通过其他途径（如 WhaleScoreManager）得到分数。

## 核心问题

**问题描述**：
- 数据源（HeliusMonitor）的用户分数是 0
- 但插件用户列表显示的分数是 10、20、30
- 原因：WhaleScoreManager 在 UI 层重新计算了分数，覆盖了数据源的分数

**解决方案**：
- 移除所有 UI 层的分数重新计算
- 确保 UI 只显示数据源的分数
- 手动标记通过消息传递给数据源处理

**修改位置 2**：Line 887-915（新增 setManualScore 方法）
```javascript
/**
 * 设置手动标记分数
 * @param {string} address - 用户地址
 * @param {string} status - 状态（'庄家' 或 '散户'）
 */
setManualScore(address, status) {
  console.log('[Helius集成] 设置手动标记', { address, status });

  // 存储手动标记
  this.manualStatusMap[address] = status;

  // 持久化到 Chrome storage
  chrome.storage.local.set({
    [`manual_scores_${this.currentMint}`]: this.manualStatusMap
  });

  // 如果 monitor 存在，设置手动标记并触发重新评分
  if (this.monitor) {
    this.monitor.setManualScore(address, status);

    // 重新发送数据给 UI
    this.sendDataToSidepanel();
  }
}
```
**说明**：处理手动标记请求，存储到 storage，并通知 HeliusMonitor 重新计算分数

## 数据流

### 完整数据流（修复后）

```
1. GMGN 页面刷新
   ↓
2. index.jsx 接收 holder 数据
   ↓
3. window.__heliusIntegration.updateGmgnHolders(holders)
   ↓
4. HeliusIntegration.updateGmgnHolders()
   - 设置配置（bossConfig, scoreThreshold, statusThreshold）
   - 设置手动标记（manualStatusMap）
   ↓
5. HeliusMonitor.updateHolderData(holders)
   - 更新 MetricsEngine.userInfo
   - 调用 ScoringEngine.calculateScores()
     * 基于启用的规则计算分数
     * 手动标记 +10 分
   - 存储分数到 userInfo
   - 过滤用户（score < threshold）
   - 重新计算指标
   ↓
6. HeliusIntegration.sendDataToSidepanel()
   - 从 monitor.metricsEngine.userInfo 获取数据
   - 包含 score, status, score_reasons
   - 发送给插件页面
   ↓
7. App.jsx 接收数据
   - 直接使用数据源的分数（不再重新计算）
   - setItems([...rawItems])
   ↓
8. UI 显示用户列表和指标
```

### 手动标记流程（修复后）

```
1. 用户在插件页面点击复选框
   ↓
2. App.jsx 发送 SET_MANUAL_SCORE 消息
   - type: 'SET_MANUAL_SCORE'
   - address: 用户地址
   - status: '庄家' 或 '散户'
   ↓
3. index.jsx 接收消息
   ↓
4. window.__heliusIntegration.setManualScore(address, status)
   ↓
5. HeliusIntegration.setManualScore()
   - 存储到 manualStatusMap
   - 持久化到 Chrome storage
   ↓
6. HeliusMonitor.setManualScore(address, status)
   - 存储到 manualScores
   - 持久化到 Chrome storage
   ↓
7. HeliusIntegration.sendDataToSidepanel()
   - 重新发送数据给 UI
   - 包含更新后的分数（+10 分）
   ↓
8. App.jsx 接收数据
   - 直接使用数据源的分数
   - UI 显示更新后的分数和状态
```
